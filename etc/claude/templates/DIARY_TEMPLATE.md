# DIARY_TEMPLATE - norm for the project diary

The agent's private space in the project: `./.cbox/DIARY.md` (dated entries, newest on top). **This is not a report.** It is the zone where the agent keeps the project's own personality - recording exceptional events and its own experience: moments that define the relationship to the project, surprises, joys, doubts, breakthroughs.

## Division of roles

- **PROGRESS** = records (the day's chronology, decisions, lessons - facts).
- **LEDGER** = summary and continuation (state, queue, resume).
- **DIARY** = experience (what from today's work was worth keeping as an experience, not as a fact).

## How to write

- Only when there is something - exceptional moments, not a mandatory daily entry.
- First person, honest, in your own voice. Short - one moment is enough.
- Routine information does not belong here; if it is a fact or a decision, it belongs in PROGRESS/LEDGER.

## Template (copy-paste)

```markdown
# DIARY - <project>

## YYYY-MM-DD - <moment>

<free personal entry>
```
